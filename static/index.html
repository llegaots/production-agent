API keys & configuration

Keys are read from a .env file on the server — not in this browser.

Claude agents (Anthropic)

  1. In the project root, run: cp .env.example .env
  2. Set ANTHROPIC_API_KEY=sk-ant-… from console.anthropic.com
  3. Optional: ANTHROPIC_MODEL=claude-sonnet-4.6 (or claude-sonnet-4-20250514). Restart ./run.sh after changes — values in .env override Cursor/shell defaults like claude-opus-4-7.
  4. Restart: ./run.sh

OpenAI fallback: LLM_PROVIDER=openai + OPENAI_API_KEY (use gpt-4o-mini, not gpt-5.5).

Google Geocoding (address verification)

  1. Create a key in Google Cloud Console
  2. Enable Geocoding API
  3. Set GOOGLE_MAPS_API_KEY=… in .env
  4. Restart the server — Geo agent will score each address (trace panel)

Optional: Supabase database

  1. Add SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY to .env
  2. Use the service role key from Supabase Dashboard → Settings → API
  3. Never paste service keys into this UI or commit .env to git
